# CATV 전송망 통합 조회 웹사이트

현장 스마트폰에서 CELL(셀명)과 B2C(전용선명)를 조회하고 국사, 선번정보, 평면도, 직선도를 확인하는 로컬 웹사이트입니다.

## 실행 방법

1. `웹사이트 실행.cmd`를 더블클릭합니다.
2. 서버가 백그라운드에서 실행되고 웹사이트가 자동으로 열립니다.
3. 직접 접속할 때는 `http://127.0.0.1:8000/`을 사용합니다.

터미널에서 실행할 때는 이 폴더에서 `npm start`를 입력해도 됩니다.

`index.html`을 직접 더블클릭해도 선명한 벡터 직선도를 사용하는 서버 화면으로 자동 연결됩니다. 서버가 꺼져 있으면 `웹사이트 실행.cmd`를 먼저 실행하세요.

## 계정과 인증

- 공개 회원가입은 제공하지 않습니다.
- 루트 관리자는 아이디 `admin`과 서버의 `RATIS_MASTER_KEY` 환경변수로만 인증합니다.
- `RATIS_MASTER_KEY` 값은 소스, 문서, `.env`, Git, GitHub에 저장하지 않습니다.
- 사용자 및 추가 관리자 계정은 루트 관리자 또는 기존 관리자가 관리자 화면의 `계정 생성`에서 아이디, 이름, 권한, 비밀번호를 지정합니다.
- 기존 가입 DB의 비밀번호 미설정 계정은 관리자가 `비밀번호 설정`을 완료한 뒤 로그인할 수 있습니다.
- 계정 비밀번호는 서버의 비공개 `data/auth-users.json`에 scrypt 해시로만 저장되고 이 경로는 Git에서 제외됩니다.
- 운영 서버에서 계정 DB를 영구 디스크에 둘 경우 `RATIS_AUTH_DB_PATH`에는 비밀값이 아닌 저장 파일 경로만 지정합니다.

### Vercel 프론트엔드와 Render 백엔드 운영 배포

- 루트의 `render.yaml`은 Render Starter 웹 서비스와 1GB 영구 디스크를 생성합니다.
- `RATIS_MASTER_KEY`는 Blueprint 생성 화면에서만 직접 입력하며 GitHub 변수, 소스, `.env`에는 저장하지 않습니다.
- 계정 해시는 영구 디스크의 `/var/data/auth-users.json`에 저장됩니다.
- 배포 상태 확인 주소는 `/api/health`이며 `apiVersion`이 `managed-auth-v1`이어야 합니다.
- 사용자 운영 주소는 Vercel의 `https://transmission-webapp.vercel.app/`입니다.
- Vercel에는 로그인 공개 셸만 배포하고 `/api`와 보호된 `/assets` 요청은 Render 백엔드로 전달합니다.
- Render 백엔드 주소는 `https://ratis-transmission-webapp-yjd1870.onrender.com/`이며 공유 DB와 도면 자산은 로그인 세션 없이는 제공하지 않습니다.
- GitHub Pages 주소는 정적 인증 화면을 제공하지 않고 Vercel 운영 주소로 이동합니다.

## 주요 기능

- CELL과 B2C 분리 조회
- 선번정보와 국사 평면도 조회
- 엑셀 원본 기반 벡터 직선도 표시
- 6글자 연속 일치 검색 위치 강조
- 큰 직선도 자동 확대 및 검색 위치 중앙 이동
- 스마트폰 두 손가락 확대·축소와 한 손가락 이동
- 관리자 지정 방식의 사용자·관리자 계정 생성, 비밀번호 재설정, 잠금, 삭제
- 데이터 DB, 국사 평면도 DB, B2C 전용선 DB 관리
- 관리자 DB별 전체 삭제

## 주요 파일

- `index.html`: 웹사이트 화면
- `styles.css`: PC·스마트폰 화면 디자인
- `app.js`: 조회, DB, 평면도와 직선도 기능
- `server.mjs`: 로컬 웹서버
- `assets/`: 안성국사 직선도 및 벡터 렌더링 자료

## GitHub Pages 배포

저장소의 `main` 브랜치에 파일을 올리면 `.github/workflows/pages.yml`이 웹사이트 파일만 묶어 GitHub Pages에 배포합니다.

배포 전 주의사항:

- GitHub Pages 주소는 인터넷에 공개될 수 있습니다.
- GitHub Pages는 정적 파일만 제공하므로 서버 인증 API와 관리자 지정 계정 로그인을 운영할 수 없습니다. 인증이 필요한 운영 주소는 `server.mjs`가 실행되는 HTTPS 서버로 배포해야 합니다.
- 직선도와 검색 인덱스에 현장 정보가 포함되어 있으므로 저장소 공개 범위를 확인하세요.
- 런타임 로그와 임시 작업 폴더는 `.gitignore`로 업로드에서 제외됩니다.
